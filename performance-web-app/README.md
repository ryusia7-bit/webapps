# 정신건강팀 실적 DB 앱 (PRD 기반 MVP)

브라우저(Chrome)에서 동작하는 로컬 웹앱입니다.

## 실행 방법

1. 권장 방식: 이 폴더에서 `serve-local.ps1`를 실행합니다.
2. 또는 `serve-local.cmd`를 실행합니다.
3. 브라우저가 자동으로 열리면 `http://127.0.0.1:8123/`에서 앱을 사용합니다.
4. `1. 엑셀 업로드` 탭에서 2025/2026 실적 파일을 선택합니다.
5. `병합` 또는 `연도 덮어쓰기` 모드를 선택하고 `업로드 및 파싱 시작`을 누릅니다.
6. `2. 데이터 현황` 탭에서 테이블별 적재 건수/미리보기를 확인합니다.
7. `3. 보고서 생성` 탭에서 월/분기/연간 보고서를 생성하고 CSV/PDF(인쇄)로 내보냅니다.

직접 `index.html`을 여는 방식도 브라우저에 따라 동작할 수 있지만, `parser-config.json` 로딩 경고가 날 수 있어 로컬 서버 방식이 더 안정적입니다.

## 구현 범위

- 엑셀 업로드 및 시트 유형별 파싱
  - 월별 일일실적
  - 입원현황/전체입원현황
  - 시설입소현황
  - 상담현황
  - 연계대장
  - 사례관리(주거지원/장기모니터링/집중사례관리)
  - 사망자현황
  - 정신과외 입원
  - 병원방문교육
  - 지역사회자원동원
  - 회의교육참여
  - 조치후 사례관리
  - 장기입원환자
- 파싱 오류 로그 수집(`#REF!`, 날짜 파싱 실패 등)
- IndexedDB(Dexie) 기반 로컬 DB 저장
- 병합/연도 덮어쓰기 모드 지원
- 보고서 자동 집계
  - 법인 월보고
  - 분기 통합 활동실적
  - 입원·시설입소 현황
  - 상담 통계
  - 사례관리 현황
  - 기타 관리 보고
- 권한별 민감정보 마스킹(사회복지사/팀장/법인보고담당)
- 백업(JSON 내보내기 + 일 단위 자동 스냅샷 30개 유지)

## 파일 구성

- `index.html`: UI 구조
- `styles.css`: 스타일
- `app.js`: 업로드/파싱/저장/집계/보고서 로직
- `parser-config.json`: 시트 분류/파싱 규칙 설정

## 참고

- 앱은 로컬 브라우저에서만 동작하며 서버가 필요 없습니다.
- 다만 설정 파일 로딩 때문에 개발 중에는 `serve-local.ps1` 또는 `serve-local.cmd`처럼 간단한 정적 서버를 쓰는 편이 안전합니다.
- PDF는 브라우저 인쇄 기능을 사용합니다.
