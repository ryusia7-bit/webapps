# 정신건강 척도검사 웹앱

현장에서 바로 쓸 수 있도록 만든 로컬 웹앱입니다.  
현재는 정적 페이지에 더해 로컬 Node 서버가 함께 동작하며, 계정 로그인/관리자 기능/서버 저장 결과 관리까지 포함합니다.

## 포함 척도

- PHQ-9
- GAD-7
- mKPQ
- K-MDQ
- PSS
- ISI-K
- AUDIT-K
- IES-R

## 실행 방법

```powershell
cd "H:\공유 드라이브\정신건강팀공유드라이브\정신건강팀자동화프로젝트\project-01-performance-automation\scale-screening-web-app"
.\serve-local.ps1
```

브라우저에서 `http://127.0.0.1:8134/`가 열립니다.

## 현재 기능

- 8개 척도 선택 및 현장용 자가검진
- 검사일, 담당자, 대상자 식별 메모, 현장 메모 입력
- 점수 계산과 결과 구간 표시
- 문항별 응답 상세 보기
- 이 기기 저장 및 저장 기록 다시 보기
- 로그인 계정 생성과 로그인
- 관리자 계정 기반 사용자 생성/권한 변경/활성화 관리
- 서버 저장 결과 보기/불러오기/삭제
- 현재 결과 JSON 내보내기
- 저장 기록 전체 CSV/JSON 내보내기
- Apps Script Web App을 통한 구글 시트 전송
- 척도 마스터(척도/문항/선택지) 시트 전송
- 기관명/팀명/연락 메모 기반 인쇄 헤더
- 인쇄용 결과지 출력

## 로그인과 관리자 기능

1. `.\serve-local.ps1`를 실행하면 로컬 Node 서버가 함께 뜹니다.
2. 기본 관리자 계정은 아래와 같습니다.
   - 아이디: `admin0109`
   - 비밀번호: `admin0109`
3. 서버 시작 시 위 관리자 계정이 자동으로 보장됩니다.
4. 관리자는 추가 실무자 계정을 생성할 수 있습니다.
5. 로그인한 상태에서는 결과를 `서버 저장`할 수 있고, 오른쪽 `계정 저장 결과`에서 다시 불러오거나 삭제할 수 있습니다.
6. 관리자 로그인 시 `관리자 기능` 카드가 나타나며 계정 생성/권한 변경/비밀번호 재설정/활성화 상태 변경이 가능합니다.

계정과 서버 저장 결과는 아래 로컬 폴더에 저장됩니다.

- `scale-screening-web-app/runtime-data/users.json`
- `scale-screening-web-app/runtime-data/records.json`

이 저장소는 현재 웹앱을 실행하는 컴퓨터 기준입니다.

## 구글 시트 연동 방법

1. `../google-apps-script` 폴더의 코드를 대상 구글 스프레드시트에 반영합니다.
2. 스프레드시트 메뉴 `데이터 자동화 > 척도검사 시트 준비`를 실행합니다.
3. 같은 메뉴에서 `척도검사 토큰 저장`을 실행하고 임의의 토큰을 저장합니다.
4. Apps Script에서 `배포 > 새 배포 > 유형: 웹 앱`으로 배포하고 Web App URL을 복사합니다.
5. 이 웹앱 왼쪽 `구글 시트 연동` 영역에 Web App URL을 입력합니다.
   - 토큰은 선택값이며, 비워두면 간편 모드로 동작합니다.
6. `척도 마스터 전송`을 한 번 눌러 8개 척도 원본 구조를 시트에 적재합니다.
7. `현재 결과 전송` 또는 `저장 기록 전체 전송`을 누르면 검사 결과와 척도 마스터가 함께 시트로 반영됩니다.

생성되는 시트는 기본적으로 아래 2개입니다.

- `척도검사기록`: 검사 1건당 1행
- `척도문항응답`: 문항/하위문항 응답 상세

추가로 척도 원본 구조용 시트도 생성됩니다.

- `척도마스터`: 척도 기본 정보
- `척도문항마스터`: 응답자 정보/문항/하위문항
- `척도선택지마스터`: 각 문항 선택지와 점수

## 인터넷 배포

이 프로젝트는 로컬 파일에 데이터를 저장하므로, 인터넷 서비스로 올릴 때는 `Node 서버`와 `영구 저장소`를 함께 써야 합니다.

제가 추천하는 방식은 `Railway`입니다.

1. 이 저장소를 GitHub에 연결합니다.
2. Railway에서 새 프로젝트를 만들고 GitHub 저장소를 연결합니다.
3. Start Command는 기본 `npm start`를 사용합니다.
4. `MH_SCALE_DATA_DIR` 환경변수를 영구 볼륨 경로로 지정합니다. 예: `/data`
5. Railway의 Volume을 서비스에 연결하고, 마운트 경로를 `/data`로 둡니다.
6. 배포가 끝나면 Railway가 주는 공개 도메인으로 접속합니다.

현재 서버는 `PORT`와 `HOST` 환경변수를 읽고, 데이터 경로는 `MH_SCALE_DATA_DIR`로 바꿀 수 있게 되어 있습니다.
인터넷 배포 시에는 `SESSION_COOKIE_SECURE=1`도 함께 설정하는 것을 권장합니다.

## 참고

- 결과는 참고용이며 진단을 대신하지 않습니다.
- 일부 척도는 공식 공개 페이지의 문항 구조를 기준으로 재구성했습니다.
- `self_seq=8`은 2026-03-24 확인 기준으로 `사건충격척도(IES-R)`로 내려오며, 사이트 다른 메뉴 문구와 불일치가 있습니다.
- 브라우저에서 구글 웹앱으로 직접 전송하므로, 전송 후에는 시트 반영 여부를 한 번 확인하는 운영 흐름을 권장합니다.
- 토큰을 비워두면 더 쉽게 쓸 수 있지만, Web App URL을 아는 사람은 해당 시트에 기록을 보낼 수 있습니다.
- 계정 기능은 로컬 운영용 경량 인증입니다. 외부 공개 서비스나 인터넷 공개 서버 용도로는 별도 보안 설계가 필요합니다.
