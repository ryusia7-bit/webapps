# 정신건강팀 실적 자동화 Apps Script

`원본데이터` 시트를 보존하면서 아래 시트를 자동 생성하는 Google Apps Script입니다.

- `기본정보`
- `병원기록`
- `시설기록`
- `서비스기록`
- `오류로그`
- `요약대시보드`
- `설정`

핵심 원칙은 다음과 같습니다.

- 원본 시트는 읽기만 하고 직접 수정하지 않습니다.
- 서비스 체크 열은 이벤트 행으로 분해합니다.
- `번호`는 고유키로 쓰지 않고 `고유번호`를 새로 생성합니다.
- 1차 운영은 증분 갱신이 아니라 전체 재구성 방식입니다.

추가로 `AI상담기록` 시트를 생성해 선택한 원본 행을 OpenAI로 분석하고 상담기록 형태로 정리하는 기능을 포함합니다.
- 상세 사용법: `../docs/counseling-ai-google-sheets.md`

## 사용 방법

1. 구글 스프레드시트에서 `확장 프로그램 > Apps Script`를 엽니다.
2. 이 폴더의 `.gs` 파일과 `appsscript.json` 내용을 그대로 붙여 넣습니다.
3. 저장 후 스프레드시트를 다시 열면 상단에 `데이터 자동화` 메뉴가 생깁니다.
4. 원본 데이터를 `원본데이터` 시트에 붙여 넣습니다.
   - `원본데이터`가 없더라도 기존 첫 일반 시트나 `시트1`이 있으면 그 시트를 원본으로 사용합니다.
   - **(중요) 원본 시트의 상단 2줄은 반드시 다음과 같은 "그룹 헤더(1행)"와 "상세 헤더(2행)" 구조여야 합니다.**
     - 1행(그룹): `공통`, `공통`, `공통`, `병원`, `시설`, `서비스` 등 (빈칸이면 이전 열의 그룹인 '공통'으로 간주)
     - 2행(상세): `날짜`, `담당자`, `이름`, `병원명`, `시설명`, `투약관리`, `기타` 등
     - 최소 필수 헤더: `공통_날짜`(그룹:공통, 상세:날짜)와 `공통_이름`(그룹:공통, 상세:이름)
   - 데이터는 **3행**부터 입력되어야 합니다.
5. `데이터 자동화 > 전체 재구성`을 실행합니다.
6. `요약대시보드`와 `오류로그`를 확인합니다.
7. AI 상담기록이 필요하면 `데이터 자동화 > OpenAI API 키 저장` 후 `AI 상담기록 시트 준비`를 실행합니다.

## 운영 메모

- `생년월일`은 `생년월일_원본`, `생년월일_표준화`를 둘 다 저장합니다.
- 날짜 또는 이름이 비어 있으면 `기본정보`에는 `유효성 = false`로 남기고 상세 시트 생성은 건너뜁니다.
- `서비스_기타`에 텍스트가 있으면 `서비스기록`에 `서비스유형 = 기타`로 1행이 추가됩니다.
- 같은 원본 행을 다시 반영할 때는 메뉴에서 `전체 재구성`만 다시 실행하면 됩니다.

## 로컬 작업 준비

Node.js가 설치된 PC에서는 이 폴더 안에서 `clasp`를 로컬 의존성으로 쓸 수 있게 `package.json`을 추가해 두었습니다.

1. `npm install`
2. `npx clasp login`
3. `npx clasp push`

직접 구글 Apps Script 편집기에 붙여 넣는 방식도 계속 사용할 수 있습니다.

## 척도검사 웹앱 연동

`scale-screening-web-app`의 저장 기록을 구글 시트로 적재하려면 아래 순서로 설정합니다.

1. 이 폴더의 최신 `.gs` 파일을 대상 스프레드시트의 Apps Script 프로젝트에 반영합니다.
2. 스프레드시트를 새로 열고 상단 메뉴 `데이터 자동화`를 확인합니다.
3. `데이터 자동화 > 척도검사 시트 준비`를 실행합니다.
   - 기본 시트는 `척도검사기록`, `척도문항응답`, `척도마스터`, `척도문항마스터`, `척도선택지마스터` 입니다.
4. `데이터 자동화 > 현재 시트를 척도검사 DB로 지정`을 실행합니다.
5. 필요하면 `데이터 자동화 > 척도검사 토큰 저장`을 실행해 웹앱과 공유할 토큰을 저장합니다.
   - 토큰 없이도 간편 모드로 사용할 수 있습니다.
6. Apps Script에서 `배포 > 새 배포 > 유형 선택: 웹 앱`으로 배포합니다.
   - 실행 사용자: 본인
   - 접근 권한: 링크가 있는 모든 사용자
7. 발급된 Web App URL을 `scale-screening-web-app` 왼쪽 패널의 `구글 시트 연동`에 넣습니다.
   - 토큰을 쓸 경우에만 같은 값을 입력합니다.
8. 웹앱에서 먼저 `척도 마스터 전송`을 눌러 척도 원본 구조를 적재하거나, 바로 결과 전송을 사용해도 척도 마스터가 함께 업데이트됩니다.

웹앱에서 보내는 데이터는 다음처럼 저장됩니다.

- `척도검사기록`: 검사 1건당 1행, 결과 요약과 전체 JSON 백업 포함
- `척도문항응답`: 각 문항/하위문항 응답을 행 단위로 저장
- `척도마스터`: 척도 기본 정보
- `척도문항마스터`: 응답자 정보, 문항, 하위문항 정의
- `척도선택지마스터`: 각 문항/하위문항/응답자 정보 선택지 정의

운영 중 점검이 필요하면 `데이터 자동화 > 척도검사 연동 상태 보기` 메뉴로 토큰 저장 여부와 시트 행 수를 확인할 수 있습니다.

토큰을 비워두면 설정은 단순해지지만, Web App URL을 아는 사람이 해당 시트에 기록을 보낼 수 있으니 내부 운영용일 때만 권장합니다.
