## 1. 설계 원칙

- 사용자 입력은 **최소 시트 수**에서 끝나야 한다.
- 집계와 보고는 **원천 DB 시트만 참조**해야 한다.
- 연도·월 전환은 **파라미터 변경**으로 해결해야 한다.
- 집계 수식은 **시트명 복사 의존성**을 가지지 않아야 한다.
- 관리자용 보조 시트는 숨길 수 있으나, 실무자 입력 시트는 최대한 단순해야 한다.

## 3. 이름 규칙 및 색상 규칙

### 탭 이름 규칙

- 입력 탭: 원본 업무명 그대로 유지
- 집계 탭: 보고서 목적을 드러내는 이름 사용
- 보조 탭: 접두어 없이 명확한 기능명 사용, 기본은 숨김

### 탭 색상 규칙

| 유형 | 권장 색상 |
| --- | --- |
| 입력(DB) | 파란색 |
| 집계/보고 | 초록색 |
| 설정/안내 | 회색 |
| 관리자 보조 | 노란색 또는 숨김 |

## 4. `기준` 시트 설계

`기준` 시트는 목록성 데이터와 설정값을 한 곳에서 관리한다.

### 4.1 레이아웃

| 영역 | 범위 예시 | 내용 |
| --- | --- | --- |
| 담당자 목록 | `A1:A20` | 드롭다운 원본 |
| 상담 항목 | `C1:C10` | 정신상담, 알콜상담, 기타상담 |
| 조치 항목 | `D1:D20` | 자의입원, 응급입원 등 |
| 사례관리 항목 | `E1:E10` | 외래진료, 병원방문 등 |
| 설정 키/값 | `G1:H10` | 기준연도, 기본월, 관리자 메일 등 |

### 4.2 권장 이름정의

| 이름 | 참조 범위 | 용도 |
| --- | --- | --- |
| `workers` | `기준!A2:A` | 담당자 드롭다운 |
| `counseling_items` | `기준!C2:C` | 항목 검증 |
| `action_items` | `기준!D2:D` | 항목 검증 |
| `case_items` | `기준!E2:E` | 항목 검증 |
| `cfg_report_year` | `기준!H2` | 기준 연도 |
| `cfg_default_month` | `기준!H3` | 기본 월 |
| `cfg_admin_email` | `기준!H4` | 관리자 이메일 |

## 5. `도움말` 시트 설계

`도움말` 시트에는 아래 내용을 반드시 포함한다.

- 어디에 입력해야 하는지: `일일활동DB`만 직접 입력
- 입력 금지 대상: 집계 시트, 수식 셀
- 날짜 입력 예시: `2026-03-16`
- 잘못된 예시: `3/16`, 담당자 직접 타이핑, 음수 입력
- 문의 채널: 운영 담당자 연락처 또는 메일

## 6. `일일활동DB` 시트 설계

### 6.1 기본 구조

| 열 | 필드명 | 입력 방식 | 비고 |
| --- | --- | --- | --- |
| A | 날짜 | 직접 입력 | 날짜만 허용 |
| B | 담당자 | 드롭다운 | `workers` 참조 |
| C | 정신상담 | 직접 입력 | 0 이상 정수 |
| D | 알콜상담 | 직접 입력 | 0 이상 정수 |
| E | 기타상담 | 직접 입력 | 0 이상 정수 |
| F | 자의입원 | 직접 입력 | 0 이상 정수 |
| G | 응급입원 | 직접 입력 | 0 이상 정수 |
| H | 보호진단 | 직접 입력 | 0 이상 정수 |
| I | 시설입소(노숙영역) | 직접 입력 | 0 이상 정수 |
| J | 시설입소(타복지영역) | 직접 입력 | 0 이상 정수 |
| K | 지원주택 | 직접 입력 | 0 이상 정수 |
| L | 주거지원 | 직접 입력 | 0 이상 정수 |
| M | 물품제공 | 직접 입력 | 0 이상 정수 |
| N | 기타(귀가·외래등) | 직접 입력 | 0 이상 정수 |
| O | 외래진료 | 직접 입력 | 0 이상 정수 |
| P | 병원방문 | 직접 입력 | 0 이상 정수 |
| Q | 주거지방문 | 직접 입력 | 0 이상 정수 |
| R | 투약관리 | 직접 입력 | 0 이상 정수 |
| S | 기타(척도·내방) | 직접 입력 | 0 이상 정수 |
| T | 입력자 | 자동 | Apps Script |
| U | 입력일시 | 자동 | Apps Script |

### 6.2 필수 설정

- 1행 헤더 고정
- 전체 필터 활성화
- A열 표시 형식: `yyyy-mm-dd`
- C:S 데이터 유효성: 숫자, 0 이상
- B열 데이터 유효성: 목록만 허용
- 조건부 서식:
  - 날짜 또는 담당자 비어 있음: 연노랑
  - 동일 날짜·담당자 중복: 연빨강
  - 미래 날짜 입력: 주황

### 6.3 권장 보호 정책

- A:S는 팀원 편집 허용
- T:U는 관리자 및 스크립트만 수정

## 7. `일일활동정규화` 보조 시트 설계

이 시트는 사용자 입력용이 아니라 보고 수식을 단순화하기 위한 보조 구조다.

### 목적

- `SUMIFS`를 세부 항목 기준으로 단순화
- 향후 피벗/대시보드 확장 용이성 확보
- wide 구조와 long 구조의 장점을 동시에 활용

### 권장 열 구조

| 열 | 필드명 | 설명 |
| --- | --- | --- |
| A | 날짜 | `일일활동DB`의 날짜 |
| B | 담당자 | `일일활동DB`의 담당자 |
| C | 항목구분 | 상담 / 조치 / 사례관리 |
| D | 세부항목 | 정신상담, 자의입원 등 |
| E | 건수 | 해당 건수 |

### 생성 방식

- 우선안: 배열수식 또는 보조 수식으로 생성
- 대안: Apps Script로 `일일활동DB` 변경 시 재생성

## 8. `일일실적_월별뷰` 시트 설계

### 8.1 상단 제어부

| 셀 | 용도 | 예시 |
| --- | --- | --- |
| B1 | 기준연도 | `=cfg_report_year` |
| B2 | 기준월 | `=cfg_default_month` |
| B3 | 월 시작일 | `=DATE(B1,B2,1)` |
| B4 | 월 말일 | `=EOMONTH(B3,0)` |

### 8.2 본문 구조

- 날짜별 블록 1개당 10행
- 구성: 담당자 8행 + 일일누계 1행 + 총누계 1행
- 31일 최대 기준으로 레이아웃을 잡고, 해당 월 이후 날짜는 공백 처리

### 8.3 수식 예시

예: 특정 날짜/담당자의 정신상담

```gs
=IF($A6="","",SUMIFS(일일활동DB!$C:$C,일일활동DB!$A:$A,$A6,일일활동DB!$B:$B,$B6))
```

예: 일일누계

```gs
=SUM(C6:C13)
```

예: 총누계

```gs
=SUM(C$6:C14)
```

## 9. 보고 시트 설계 원칙

### 공통 원칙

- 월/분기 기준 셀을 상단에 둔다.
- 직접 시트명을 참조하지 않고 날짜 범위로 집계한다.
- 합계 셀은 수동 입력 금지, 보호 설정 필수

### `법인월보고용`

- 월별 12칸 + 분기합 + 연누계 구조 유지
- 상담/조치/사례관리 섹션 분리
- 가능하면 원천 DB별 참조 범위를 블록 단위로 정리

### `분기보고용`

- 분기 시작월 셀을 두고 분기 범위를 계산한다.
- 연계경로 항목은 `연계대장` 매핑표가 확정된 뒤 반영한다.

### `입원실적`

- `입원형태`, `성별`, `퇴원후연계`별 블록을 분리한다.

### `연계실적`

- 월/주 파라미터를 모두 두거나, 월 기준 + 주 블록 방식으로 구성한다.

## 10. 보호 범위 정책

| 탭 | 보호 범위 | 예외 |
| --- | --- | --- |
| 기준 | 전체 | 관리자만 |
| 도움말 | 전체 | 관리자만 |
| 일일활동DB | T:U, 헤더 | 실무 입력 범위는 편집 허용 |
| 케이스 DB | 헤더/수식열 | 나머지 입력 범위 허용 |
| 집계/보고 탭 | 전체 | 관리자만 |
| 보조 탭 | 전체 | 관리자만 |

## 11. 권장 검증 셀

각 보고 시트에는 최소 1개 이상의 검증 셀을 둔다.

| 시트 | 검증 내용 | 예시 |
| --- | --- | --- |
| 법인월보고용 | 월합 = 세부항목 합 | 차이 0 |
| 입원실적 | 유형별 합 = 월 전체 입원 수 | 차이 0 |
| 연계실적 | 주합 = 월합 | 차이 0 또는 예외 설명 |
| 조치후사례관리(집계) | 세부 항목 합 = 전체 | 차이 0 |

## 12. 권장 추가 개선
